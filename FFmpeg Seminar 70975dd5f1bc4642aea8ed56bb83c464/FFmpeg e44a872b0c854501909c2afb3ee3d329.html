How to use CLI and specify input and output

I apologize for misunderstanding your request earlier. Here are two examples of basic FFmpeg commands without any codecs or filters:

  1. Copying video and audio streams to a new file:
ffmpeg -i input.mp4 -c copy output.mp4

This command copies both the video and audio streams from the input file input.mp4 to a new file named output.mp4, without transcoding or altering the streams in any way. This is useful for simply re-muxing a video file into a different container format.

  1. Extracting audio from a video file:
ffmpeg -i input.mp4 -vn -c:a copy output.mp3

This command extracts only the audio stream from the input file input.mp4 and saves it as an MP3 file named output.mp3, without transcoding or altering the audio stream in any way. The -vn option tells FFmpeg to disable video recording.

Using codecs - Converting a video to the H.264 codec

  1. Open a terminal or command prompt window and navigate to the folder where your input video file is located.
  1. Use the following command to convert the video to the H.264 codec:
ffmpeg -i input_video.avi -c:v libx264 -preset slow -crf 22 -c:a copy output_video.mp4

Replace "input_video.avi" with the filename of your input video file, and "output_video.mp4" with the desired name of your output video file. This command tells FFmpeg to re-encode the video stream using the H.264 codec, and copy the audio stream without re-encoding.

  1. Wait for FFmpeg to complete the video processing. This may take a few minutes to several hours depending on the length and complexity of your video file.
  1. Once the video has been generated, use a video player or editing software to view and edit the final video as needed.

Tips:

  • The "-preset slow" option in the FFmpeg command above specifies a slower but higher quality encoding process. You can adjust this option to a faster or slower preset depending on your desired balance of speed and quality.
  • The "-crf 22" option in the FFmpeg command above specifies the video quality level, with a lower number indicating higher quality. You can adjust this option to a higher or lower value depending on your desired video quality

Applying filters - Cropping a video and adding a watermark

Instructions:

  1. Open a terminal or command prompt window and navigate to the folder where your input video file and watermark image file are located.
  1. Use the following command to crop the video and add a watermark:
ffmpeg -i input_video.mp4 -i watermark.png -filter_complex "[0:v]crop=640:480:0:0[v];[v][1:v]overlay=10:10" -c:a copy output_video.mp4

ffmpeg -i input_file.mp4 -i watermark.png -filter_complex "[0:v][1:v]overlay=W-w-10:H-h-10" -c:a copy output_file.mp4

Replace "input_video.mp4" with the filename of your input video file, "watermark.png" with the filename of your watermark image file, and "output_video.mp4" with the desired name of your output video file. This command tells FFmpeg to crop the video to a resolution of 640x480 pixels starting from the top-left corner, and add a watermark image at position (10,10) on the video using the "overlay" filter.

  1. Wait for FFmpeg to complete the video processing. This may take a few minutes to several hours depending on the length and complexity of your video file.
  1. Once the video has been generated, use a video player or editing software to view and edit the final video as needed.

Tips:

  • The "crop" filter in the FFmpeg command above takes four parameters: the width and height of the cropped region, and the x and y coordinates of the top-left corner of the cropped region.
  • The "overlay" filter in the FFmpeg command above takes two parameters: the x and y coordinates of the top-left corner of the overlay image on the video. You can adjust these parameters to position the watermark image as desired.

Compression Techniques

  • Bit rate reduction:
  1. Convert the video to H.264 or HEVC format with a lower bit rate:
ffmpeg -i input.mp4 -c:v libx264 -b:v 5M output.mp4

This command uses the H.264 codec with a bit rate of 5 Mbps to convert the input video to a lower bit rate output.

  1. Use the VP9 or AV1 codec for better compression:
ffmpeg -i input.mp4 -c:v libvpx-vp9 -b:v 5M output.webm

This command uses the VP9 codec to convert the input video to a lower bit rate output.

  1. Remove unnecessary audio and subtitle tracks:
ffmpeg -i input.mp4 -map 0:v -map 0:a:0 -c:v libx264 -b:v 5M output.mp4

This command removes all audio and subtitle tracks except the first audio track and converts the input video to a lower bit rate output using the H.264 codec with a bit rate of 5 Mbps.

  • Resolution reduction:
  1. Downscale the video to 720p resolution:
ffmpeg -i input.mp4 -vf scale=1280:720 output.mp4

This command uses the scale filter to downscale the input video to 720p resolution.

  1. Crop the video to remove unnecessary portions of the frame:
ffmpeg -i input.mp4 -vf crop=1280:720:100:100 output.mp4

This command uses the crop filter to remove a 100 pixel border from each side of the input video and converts it to 720p resolution.

  • Frame rate reduction:
  1. Reduce the frame rate of the video to 30 fps:
ffmpeg -i input.mp4 -r 30 output.mp4

This command uses the r flag to reduce the frame rate of the input video to 30 fps.

  1. Use interlaced encoding:
ffmpeg -i input.mp4 -c:v libx264 -preset fast -tune film -flags +ildct+ilme -top 1 -b:v 5M output.mp4

This command uses interlaced encoding with the H.264 codec and a bit rate of 5 Mbps to reduce the overall frame rate while maintaining a similar perceived quality.

Demonstrate how to use FFmpeg to transcode video files to different formats, such as H.264, HEVC

  1. Transcoding to H.264:
ffmpeg -i input_file.mp4 -c:v libx264 -preset veryfast -crf 20 -c:a copy output_file.mp4

This command transcodes the input video file to H.264 format using the libx264 codec with a very fast encoding preset and a constant rate factor (CRF) of 20. The audio stream is copied without transcoding.

  1. Transcoding to HEVC:
ffmpeg -i input_file.mp4 -c:v libx265 -preset medium -crf 25 -c:a copy output_file.mp4

This command transcodes the input video file to HEVC format using the libx265 codec with a medium encoding preset and a CRF of 25. The audio stream is copied without transcoding.

  1. Transcoding to H.264 with audio transcoding:
ffmpeg -i input_file.mp4 -c:v libx264 -preset medium -crf 23 -c:a aac -b:a 128k output_file.mp4

This command transcodes the input video file to H.264 format using the libx264 codec with a medium encoding preset and a CRF of 23. The audio stream is transcoded to AAC format with a bit rate of 128 kbps.

  1. Transcoding to HEVC with audio transcoding:
ffmpeg -i input_file.mp4 -c:v libx265 -preset slow -crf 28 -c:a aac -b:a 96k output_file.mp4

This command transcodes the input video file to HEVC format using the libx265 codec with a slow encoding preset and a CRF of 28. The audio stream is transcoded to AAC format with a bit rate of 96 kbps.
